    # Docker

    # 免sudo使用docker

    1.如果还没有 docker group 就添加一个

    sudo groupadd docker

    2.将用户加入该 group 内。然后退出并重新登录就生效啦

    sudo gpasswd -a ${USER} docker

    3.重启 docker 服务

    sudo service docker restart

    4.切换当前会话到新 group 或者重启 X 会话

    newgrp - docker

    # 启动容器

    docker run -idt ubuntu

    # 查看启动中的容器

    docker ps

    # 查看所有容器（启动的和停止）

    docker ps -a

    # 进入容器

    docker exec -it [容器id]

    # 终止容器

    docker stop [容器id]

    # 删除容器

    • 需要先停止容器
    docker rm [容器id]
    • 直接删除运行中的容器
    docker rm -f [容器id]

    清理所有处于终止状态的容器

    docker rm $(docker ps -a -q)
docker rm $(docker container ls -f "status=exited" -q)

    # 下载镜像

    docker pull --help
docker pull [选项] [Docker Registry地址]<仓库名>:<标签>

    下载ubuntu 14.04

    docker pull ubuntu:14.04

    # 运行一个容器，指定Shell为 bash

    docker run -it --rm ubuntu:14.04 bash
    • docker run 运行容器的命令
    • it 这是2个参数的组合命令
    • -i 交互操作
    • -t 终端
    • -rm 容器退出后随即删除
    • ubuntu:14.04 指用ubuntu:14.04镜像为基础启动容器
    • bash 这里是指定运行Shell的bash

    # Dockerfile 定制镜像

    1.创建Dockerfile文件

    2.写入内如

    FROM nginx
RUN echo 'Hello, Docker!!' > /usr/share/nginx/html/index.html

    3.构建镜像

    docker build [options] <context url>
docker build -t nginx:v1 .

    注意后面加了 ‘.’ 意思是在当前目录

    # 直接用Git仓库进行构建

    docker build https://github.com/twang2218/gitlab-ce-zh.git#:8.14

    这行命令指定了构建所需的 Git repo，并且指定默认的 master 分支，构建目录为 /8.14/，然后 Docker 就会自己去 git clone 这个项目、切换到指定分支、并进入到指定目录后开始构建。

    # tar 压缩包构建

    docker build http://server/context.tar.gz

    # 从标准输入中读取 Dockerfile 进行构建

    docker build - < Dockerfile
or
cat Dockerfile | docker build -

    # Docker 构建一个nodejs服务

    创建一个目录

    mkdir docker-node && cd docker-node

    创建文件

    touch app.js
touch package.json
touch Dockerfile

    写入内容 Dockerfile

    FROM node:6.10.2-alpine
ADD . /server/www/
WORKDIR /server/www/
RUN cd /server/www && npm install
EXPOSE 3001
CMD ["node","server.js"]

    package.json

    {
  "name": "docker-node",
  "private": true,
  "version": "1.0.0",
  "description": "Node.js with docker",
  "author": "you@email.com",
  "dependencies": {
    "express": "3.2.4"
  }
}

    app.js

    var express = require('express');
var PORT = 3001;
var app = express();

app.get('/', function (req, res) {
  res.send('Hello World\n');
});

app.listen(PORT)
console.log('Running on http://127.0.0.1:' + PORT);
